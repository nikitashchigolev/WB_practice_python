Имеется таблица, которая должна состоять из 1000 строк и содержать данные о кликах пользователей на всем пути использования сайта (по всем функциональным элементам): user_id, event_date время совершения действия, session_id (считаем как одно посещение сайта), element_name (название элемента, по которому было совершено действие), event_id (PK), event_profit (int в рублях), ver (версия сайта) и platform сайта (для какого устройства).
CREATE TABLE UserClicks (
    event_id INT PRIMARY KEY,
    user_id INT,
    event_date DATE,
    session_id INT,
    element_name VARCHAR(255),
    event_profit INT,
    ver VARCHAR(255),
    platform VARCHAR(255)
);
В столбцах должны быть следующие значения:
element_name = ['Advert Banner', 'Work and Partnership', 'Product Card', 'Buy Now']
ver = ['Global', 'Original'], где 'Global' - https://global.wildberries.ru/,  'Original' - https://www.wildberries.ru/
platform = ['Desktop','Mobile']
Исходя из проведенного анализа сайтов различных версий и платформ (в pdf файле), напишем условия для генерации данных:
1. В рамках одной сессии (для одного пользователя может быть больше одной записи с одним и тем же session_id) для одного пользователя разброс значений дат event_date должен быть не больше часа.
2. Для element_name = 'Advert Banner' и ver = 'Original' и (platform = 'Desktop' или 'Mobile' ) event_profit должен быть в среднем выше, чем у остальных записей.
3. Для element_name = 'Work and Partnership' и ver = 'Original' и platform = 'Desktop' event_profit в среднем должен быть выше, чем у остальных записей.
4. Для element_name =  'Product Card' и (ver = 'Global' и platform = 'Desktop' или 'Mobile') или (ver = 'Original' и platform = 'Desktop') event_profit должен быть в среднем выше, чем у остальных записей.
5. Для element_name = 'Buy Now' и ver = 'Original' и (platform = 'Desktop' или 'Mobile') event_profit должен быть в среднем выше, чем у остальных записей.

Метрики:
1. Конверсия в целевое действие показывает, как много пользователей выполнили конкретное целевое действие (например, покупка или регистрация) относительно общего числа пользователей, посетивших сайт. Например, в нашем случае действие напрямую связано с названием элемента, поэтому используем его для определения конверсии в покупку.
2. Средний доход с пользователя (ARPU - Average Revenue Per User)- это то метрика, которая оценивает средний доход, полученный с каждого пользователя. ARPU может помочь определить, какая версия и платформа приводит к более высокому среднему доходу с пользователя.
3. Средний доход с сессии (ARPS - Average Revenue Per Session) оценивает средний доход с каждой сессии на сайте.
4. Средний доход с элемента (ARPE - Average Revenue Per Element) оценивает средний доход, полученный с каждого элемента сайта (например, Advert Banner, Product Card, и другие).

Исходя из результов, можно сделать вывод о том, что сайт Original (https://www.wildberries.ru/) Desktop является наиболее выгодным с точки зрения бизнеса. Для максимизации полученных метрик в будущем необходимы проведение A/B-тестирования и добавление в новую версию сайта фич (функциональных элементов) остальных рассматриваемых версий, которые могли бы потенциально улучшить взаимодействие пользователя с продуктом.
